# Security Policy

## Supported Versions

We support the following versions of the Performance Optimisation plugin:

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | :white_check_mark: |
| 1.5.x   | :x:                |
| < 1.5   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in Performance Optimisation, please report it responsibly:

1.  **Do NOT open a public issue** on GitHub.
2.  **Email us** at [security@example.com](mailto:security@example.com) (replace with actual email if available) or contact the author directly via [WordPress.org profile](https://profiles.wordpress.org/nileshkanzariya/).
3.  Include details about the vulnerability and steps to reproduce it.

We will acknowledge your report within 48 hours and work with you to fix the issue.

## Security Best Practices

*   Keep the plugin updated to the latest version.
*   Use strong passwords for your WordPress admin accounts.
*   Limit login attempts.
*   Keep your PHP and WordPress versions up to date.
