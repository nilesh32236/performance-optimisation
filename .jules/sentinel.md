## 2024-05-18 - Missing Input Sanitization on REST API Settings Import
**Vulnerability:** The `import_settings` REST endpoint in `class-rest.php` allowed saving raw, unsanitized JSON input directly into the `wppo_settings` database option, whereas the individual `update_settings` endpoint properly sanitized settings. This allowed for potential Cross-Site Scripting (XSS) via maliciously crafted options being rendered unescaped on the frontend or backend.
**Learning:** Even endpoints restricted by `current_user_can('manage_options')` and a valid REST nonce need comprehensive input sanitization. Importers are frequent targets for privilege escalation or maintaining persistence. Missing parity between endpoints (e.g. `import` vs. `update`) is a common pattern for vulnerabilities.
**Prevention:** Always ensure any method that writes complex structures to the database utilizes a robust, recursive sanitization method (`sanitize_settings_recursively`) instead of trusting authenticated user input directly. Maintain parity of input validation across all write interfaces for a single data object.
